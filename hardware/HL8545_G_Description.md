# HL8545/G 说明文档

> 资料来源：`HL8545_G Datasheet_REV1.1_20241212.pdf`，器件手册版本 `rev 1.1 Dec.2024`。  
> 本文面向 ECAS 阀驱动/高边开关集成使用，重点整理器件功能、硬件连接、诊断方式、保护行为和本工程软件配置要点。

## 1. 器件概述

HL8545/G 是一款车规四通道高边功率开关，每个通道内部集成 NMOS 功率管、栅极驱动、电荷泵、电流限制、开路检测、过载检测、温度保护和电流采样诊断功能。器件用于把电池电源 `VIN` 高边切换到负载 `OUTPUTx`，适合驱动小功率灯、继电器、线圈、电磁阀、电机、加热器以及其他阻性、感性或容性负载。

器件没有 SPI 或寄存器接口。其控制与诊断全部通过离散 IO 和模拟诊断脚完成：

- `EN0..EN3`：分别控制 4 个输出通道开关。
- `SEn`：使能 `SNS` 诊断输出。
- `SEL0/SEL1`：选择当前被 `SNS` 监控的输出通道。
- `SNS`：输出负载电流比例电流，也可在故障时输出高电平诊断电压。
- `nFLTR`：==低有效故障复位/模式配置脚==，用于故障解锁，并决定热保护场景下锁存或自动恢复行为。

HL8545 与 HL8545G 的主要差异在反接保护：

- `HL8545`：反接电池时需要外部 GND 防反网络，通常使用 GND 串联/并联二极管与电阻方案。
- `HL8545G`：内部集成从 GND 到 VIN 的反向电流阻断能力，可不加外部 GND 二极管；但输出级反向电流仍需要由负载或外部电路限制。

## 5. 基本工作方式

### 5.1 输出控制

每个输出通道由对应 `ENx` 单独控制：

| `ENx` | `OUTPUTx` 行为      |
| ----- | ----------------- |
| Low   | 输出关闭，进入 OFF 态诊断条件 |
| High  | 输出打开，向负载供电        |

当通道打开时，器件会监控通道电流、温度和短路状态。过载或短路到地会触发限流；持续故障可能进一步触发热摆幅或热关断。

### 5.2 正常模式与待机模式

器件有正常模式和待机模式。满足以下条件并经过待机消隐时间后进入待机：

- `EN0..EN3 = Low`
- `nFLTR = Low`
- `SEn = Low`
- `SEL0 = Low`
- `SEL1 = Low`

==待机消隐时间 `tD_STB` 典型 `200 us`，最大 `500 us`。任一控制脚被拉高时，器件返回正常模式。

### 5.3 `nFLTR` 的作用

`nFLTR` 是低有效输入，兼容 3.3 V/5 V 逻辑，具有两个核心用途：

1. ==故障锁存复位：发生锁存类故障后，将 `nFLTR` 从高拉低并保持至少 `tLATCH_RST`，可解锁输出。
2. 热保护模式选择：`nFLTR=High` 时故障趋向锁存；`nFLTR=Low` 时热摆幅/热关断后可自动恢复。

关键时间：`tLATCH_RST` 最小 `10 us`，最大 `20 us`。本工程当前配置 `VALVE_HL8545_FRST_PULSE_US = 20U`，符合手册上限要求。

## 6. `SNS` 电流采样与诊断

### 6.1 诊断通道选择

`SNS` 只能同时输出一个被选通通道的信息。选择关系如下：

| `SEn` | `SEL1` | `SEL0` | 被诊断通道 | 正常 ON 态 `SNS` |
| --- | --- | --- | --- | --- |
| Low | X | X | 无 | 高阻 |
| High | Low | Low | Channel 0 | `ISNS = IOUT0 / K` |
| High | Low | High | Channel 1 | `ISNS = IOUT1 / K` |
| High | High | Low | Channel 2 | `ISNS = IOUT2 / K` |
| High | High | High | Channel 3 | `ISNS = IOUT3 / K` |

当被选通通道发生过载、OFF 态开路、OFF 态短路到电池、热摆幅或热关断等故障时，`SNS` 不再作为线性电流采样，而是被拉到故障诊断电压 `VSENSEH`。

### 6.2 电流计算

HL8545/G 使用电流镜输出采样电流：

```text
ISNS = IOUT / K
VSNS = ISNS x RSNS
IOUT = VSNS x K / RSNS
```

其中：

- `K`：电流采样比例，典型值 `1150`。
- `RSNS`：`SNS` 到 GND 的采样电阻。
- `VSNS`：MCU ADC 采到的 `SNS` 电压。

本工程配置：

```c
#define VALVE_HL8545_SENSE_RESISTOR_OHM           (1000U)
#define VALVE_HL8545_CURRENT_SENSE_RATIO          (1150U)
```

因此当前工程的换算关系为：

```text
IOUT_mA = VSNS_mV x 1150 / 1000
        = VSNS_mV x 1.15
```

示例：

| `VSNS` | 估算负载电流 |
| ---: | ---: |
| `100 mV` | `115 mA` |
| `500 mV` | `575 mA` |
| `1000 mV` | `1150 mA` |
| `4000 mV` | `4600 mA` |

设计注意：

- 正常线性采样时，应确保 `VSNS` 处于 `0 V ~ 5 V` 区间。
- 当 `SNS` 达到 `VSENSEH` 或 ADC 接近高电平饱和时，应优先按故障处理，而不是继续换算电流。
- `SNS` 不能悬空；不用时通过 `1 kOhm` 接地。

### 6.3 诊断时序

| 场景 | 典型/最大时间 | 说明 |
| --- | --- | --- |
| `SEn` 上升到 `SNS` 有效，`RL=6.5 Ohm` | `10 us` | `SNS` 到 90% |
| `SEn` 上升到 `SNS` 有效，`RL=100 Ohm` | `60 us` | 轻载时更慢 |
| `SEn` 下降到 `SNS` 关闭，`RL=6.5 Ohm` | `2 us` | `SNS` 到 10% |
| `SEn` 下降到 `SNS` 关闭，`RL=100 Ohm` | `60 us` | 轻载时更慢 |
| `ENx` 上升后 `SNS` 有效 | `120 us ~ 250 us` | 从输出打开到采样稳定 |
| `ENx` 下降后 OFF 态诊断有效 | `300 us ~ 500 us` | 开路/短电池诊断消隐 |
| `SEn` 上升后 OFF 态开路指示有效 | `250 us` | 从诊断使能到有效 |
| OFF 态 `VOUT` 上升后诊断延迟 | `2.5 us ~ 18 us` | `VIN - VOUT < VOL` 时触发 |

软件采样建议：

- 切换 `SEL0/SEL1` 后，不要立即读取 ADC；应留出 `SNS` 稳定时间。
- 新打开通道后建议至少等待 `250 us` 再进行电流有效性判定。
- 关闭通道后做开路/短电池诊断，建议等待 `500 us` 级别消隐时间。
- 对 PWM 驱动的阀类负载，采样点应避开开关沿和限流消隐阶段。

## 7. 故障诊断与保护

### 7.1 故障总览

下表假设 `SEn=High` 且对应通道已被 `SEL0/SEL1` 选中：

| 状态 | `ENx` | `OUTx` 表现 | `nFLTR` | 判据 | `SNS` 表现 | 输出行为 |
| --- | --- | --- | --- | --- | --- | --- |
| 正常关闭 | Low | Low | X | 无故障 | 约 0 或高阻相关状态 | 关闭 |
| 正常打开 | High | High | X | 无故障 | 线性电流采样 | 打开 |
| 过载/短路到 GND | High | 被限流/拉低 | High | 触发电流限制 | `VSENSEH` | 锁存关闭 |
| 过载/短路到 GND | High | 被限流/拉低 | Low | 触发电流限制 | `VSENSEH` | 自动恢复 |
| OFF 态开路 | Low | `VIN - VOUT < VOL` | X | 开路阈值命中 | `VSENSEH` | 输出保持关闭 |
| OFF 态短路到电池 | Low | `VOUT` 被外部拉高 | X | 与开路同类判据 | `VSENSEH` | 输出保持关闭 |
| 热关断 | High | 关闭 | High | `Tj > TTSD` | `VSENSEH` | 锁存关闭 |
| 热关断 | High | 关闭/恢复循环 | Low | 温度回落后恢复 | `VSENSEH` 后恢复 | 自动恢复 |
| 热摆幅 | High | 关闭 | High | 温升过快 | `VSENSEH` | 锁存关闭 |
| 热摆幅 | High | 关闭/恢复循环 | Low | 温差回落后恢复 | `VSENSEH` 后恢复 | 自动恢复 |

### 7.2 过载和短路到地

ON 态下，如果 `OUTPUTx` 短路到 GND 或负载过重，器件进入限流。限流高阈值 `ILIMH` 在 `VIN=13 V` 时典型约 `19 A`，最大约 `22 A`。热循环阶段限流会折返到 `ILIML`，典型约 `8 A`。

当被选中通道处于限流状态，`SNS` 输出 `VSENSEH`。如果故障消除，自动恢复模式下器件可恢复输出；如果故障持续，温度保护会进一步关闭功率管。

### 7.3 OFF 态开路和短路到电池

OFF 态下，内部弱上拉连接到 `OUTx`，用于开路/短路到电池检测。当满足：

```text
VIN - VOUT < VOL
```

器件判断为 OFF 态开路或短路到电池。`VOL` 阈值典型约 `2.5 V`，范围约 `1.5 V ~ 3 V`。这两类故障在 `SNS` 上的表现相同，均为 `VSENSEH`，需要结合输出状态、负载连接和系统电压进一步区分。

### 7.4 热关断与热摆幅

器件同时具备两类温度保护：

- 绝对温度热关断：当结温超过 `TTSD` 后关闭输出。`TTSD` 范围约 `150 C ~ 200 C`。
- 热摆幅保护：当功率管温度相对逻辑区域温度快速上升，温差超过 `TSW` 后关闭输出。`TSW` 典型约 `60 C`。

热保护迟滞 `THYST` 典型约 `15 C`。热关断后的温度复位延迟 `tDLY_RST` 典型约 `3 ms`。

`nFLTR` 对热保护后的行为很关键：

- `nFLTR=High`：故障锁存，输出和 `SNS` 故障状态保持，直到 `nFLTR` 被拉低复位。
- `nFLTR=Low`：自动恢复，温度回落并满足延迟后尝试重新打开输出。

### 7.5 欠压保护

`VIN` 下降到 UVLO 阈值附近时，器件进入欠压保护，输出关闭。关键值：

| 参数 | 典型值 |
| --- | ---: |
| UVLO falling threshold | `3.4 V` |
| UVLO reset | `3.6 V` |
| UVLO hysteresis | `200 mV` |

系统软件应避免在 `VIN` 过低时把阀驱动故障误判为负载问题。

### 7.6 GND 丢失与供电丢失

器件支持 GND 丢失和供电丢失保护：

- GND 丢失时，无论 `ENx` 高低，输出会关闭。
- 供电丢失时，输出也会关闭。对阻性/容性负载较直接；对感性负载，电感电流可能通过 MCU IO 或外部路径续流，因此需要串联电阻、GND 防护网络或外部续流设计。

## 8. 反接与 MCU IO 保护

### 8.1 反接保护方案

可选方案如下：

| 方案 | 说明 | 适用性 |
| --- | --- | --- |
| VIN 串联防反二极管 | 保护器件和负载，但有压降和功耗 | HL8545/HL8545G 均可 |
| GND 防反网络 | 限制反接时 GND 引脚电流 | HL8545 常用 |
| GND 电阻与二极管并联 | 手册推荐方案，兼顾反接限流和感性负载尖峰泄放 | HL8545 推荐 |
| HL8545G 内部防反 | 不加外部 GND 二极管也可阻断 GND 到 VIN 反向电流 | 仅 HL8545G |

手册推荐的 GND 网络方案为 `1 kOhm` 电阻并联正向电流能力大于 `100 mA` 的二极管。若多个高边开关共用同一 GND 防护网络，需要核算总电流、地偏移和功耗。

### 8.2 MCU IO 串联电阻

在负 ISO 脉冲、感性负载供电丢失、GND 负压等场景下，HL8545/G 的 GND 或 IO 可能出现对 MCU 不利的电压。建议 MCU 与 HL8545/G 的控制脚之间串联电阻。手册对 GND 丢失保护场景推荐 `50 kOhm` 量级。

需要重点保护的 MCU 连接：

- `EN0..EN3`
- `SEn`
- `SEL0`
- `SEL1`
- `nFLTR`
- `SNS` 到 ADC 的输入路径

## 9. 感性负载关断

HL8545/G 内置功率管漏源钳位，用于感性负载关断时限制输出负压和功率管应力。关断线圈、电磁阀、继电器等负载时，电源侧能量和负载电感能量会在高边开关内部消耗。

关键参数：

| 参数 | 数值 |
| --- | ---: |
| VDS clamp / decay voltage | 典型约 `40 V` |
| 单脉冲最大开关能量 | `30 mJ`，衰减时间 `0.4 ms`，从 `Tj=150 C` 开始 |

设计建议：

- 阀、继电器、线圈类负载必须按最大电流、最大电感、最高环境温度核算关断能量。
- 如果能量超过器件能力，或关断频率较高，应增加外部续流、TVS 或其他能量释放路径。
- 软件 PWM 频率和占空比策略也会影响开关损耗与芯片温升。

### 11.4 软件诊断流程建议

推荐的单通道电流诊断流程：

1. 调用 `Valve_Diag_SelectChannel(Channel)` 选择目标通道。
2. 等待 `SNS` 稳定。若只是切换诊断通道，建议等待至少 `60 us`；若刚打开输出，建议等待至少 `250 us`。
3. 读取对应 ADC 通道的 `SNS` 电压。
4. 若 `SNS` 电压接近 `VSENSEH` 或超过软件设定故障阈值，进入故障判定分支。
5. 否则调用 `Valve_Diag_UpdateCurrent(Channel, SnsVoltage_mV)` 更新负载电流。
6. 轮询下一通道前重复上述步骤。

推荐的 OFF 态开路/短电池诊断流程：

1. 确保目标通道 `ENx=Low`。
2. 调用 `Valve_Diag_SelectChannel(Channel)`。
3. 等待 OFF 态诊断消隐时间，建议按 `500 us` 设计。
4. 读取 `SNS`。若为 `VSENSEH`，表示 OFF 态开路或短路到电池。
5. 结合 `VOUT`、负载状态、线束状态或系统诊断上下文进一步区分开路与短电池。

锁存故障恢复流程：

1. 记录故障通道和故障类型。
2. 确认系统允许重新打开该高边输出。
3. 调用 `Valve_ResetFault(Device)` 对 `nFLTR` 产生低脉冲。
4. 等待输出和 `SNS` 恢复稳定。
5. 重新采样并确认故障已消失。

## 12. 设计与调试注意事项

- `SNS` 是模拟诊断输出，不是数字故障脚；软件必须区分线性电流电压和 `VSENSEH` 故障电压。
- `SEn=Low` 时 `SNS` 为高阻，不应据此判断电流为 0。
- 多通道共用一个 `SNS` 时，ADC 采样必须和 `SEL0/SEL1/SEn` 时序同步。
- 大电容负载或冷态灯泡上电浪涌可能触发限流；超过 `tOL_BLK=0.3 ms` 后会在 `SNS` 上报故障。
- `nFLTR` 保持高电平会让部分故障锁存，适合故障需要 MCU 明确介入的系统；保持低电平会倾向自动恢复，适合允许自恢复的负载。
- 感性负载关断能量、重复开关频率和环境温度需要一起评估，不能只看单次 `30 mJ` 指标。
- HL8545 与 HL8545G 的反接外部电路不同，硬件 BOM 和 PCB 设计时必须确认实际料号。
- `NC` 脚建议悬空；若接地，应确认不会引入 GND 偏移或测试模式风险。
- MCU IO 与 HL8545/G 之间建议预留串联电阻，特别是 `ENx`、`SEn`、`SELx`、`nFLTR` 和 `SNS` 路径。
- 在低电压或供电瞬断时，应把 UVLO/供电故障和负载故障区分处理，避免误报阀线束故障。

## 13. 快速检查清单

硬件检查：

- `VIN` Exposed Pad 是否接电池电源并满足散热铜皮要求。
- `SNS` 是否有合适的下拉采样电阻，且 ADC 输入量程可覆盖 `VSENSEH`。
- `SEn/SEL0/SEL1/nFLTR` 是否有明确默认状态。
- `nFLTR` 是否由 MCU 可控，且电平策略与锁存/自动恢复需求一致。
- 实际使用的是 HL8545 还是 HL8545G，反接保护网络是否匹配。
- 感性负载关断能量是否小于器件能力，必要时是否有外部续流/钳位。

软件检查：

- `SEL0/SEL1` 通道表是否与硬件原理图一致。
- ADC 采样是否避开 PWM 边沿和 `SNS` 稳定时间。
- `VSENSEH` 是否有独立阈值判定，不与正常电流混淆。
- `Valve_ResetFault()` 低脉冲是否满足 `10 us ~ 20 us` 要求。
- OFF 态诊断是否等待足够消隐时间。
- 故障恢复是否有次数限制、冷却等待或上层安全策略。
