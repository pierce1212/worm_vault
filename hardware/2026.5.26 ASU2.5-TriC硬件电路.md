## 电机高边
![](assets/2026.5.26%20ASU2.5-TriC硬件电路/file-20260526174425587.png)
这个电路本质上是：**M+ 电压采样 + 掉电隔离 + 弱上拉诊断**。

可以等效看成：

```text
MOTOR_HSD -- R609 51k -- M+ -- Q605B(PNP开关) -- R611 15k -- adc_M+ -- R614 4.7k -- GND
                                      ^
VCC_5V_1 -- Q605A(NPN) --------------|
adc_M+ -- R613 10k -- ADC_V_M+
                 |
              C606 1nF
                 |
                GND
```

Q605 是 SMUN5312DW1T1G，一颗互补“数字三极管”，里面是 NPN + PNP，并带内部偏置电阻。按 [onsemi datasheet](https://www.part-elec.com/datasheet/on-semiconductor/SMUN5312DW1T1G.pdf) 看，它适合做这种小电流电平开关。

工作逻辑：

- **5V 上电时**：Q605A 的输入接 `VCC_5V_1`，NPN 导通，把 Q605B 的控制端拉低；Q605B 是 PNP，于是导通，把 `M+` 接到右侧分压电阻。
- **ADC 采样比例**：  
  `R611 = 15k`，`R614 = 4.7k`，所以  
  `adc_M+ ≈ M+ × 4.7 / (15 + 4.7) ≈ M+ × 0.238`  
  也就是软件里大约要乘 `4.19` 还原 M+ 电压。
- **5V 掉电时**：Q605A 关断，Q605B 也关断，`M+` 不会通过分压电阻/ADC 管脚反灌 MCU 供电，分压也不再耗电。
- **R609 51k**：这是从 `MOTOR_HSD` 到 `M+` 的弱上拉/诊断电阻，不是功率通路。若只有这条弱路径，12V 下电流约：  
  `12V / (51k + 15k + 4.7k) ≈ 170uA`  
  这和图里的红字 `170uA` 对得上。它可能用于开路、短地、HSD 状态诊断。
- **R613 + C606**：给 ADC 做低通滤波和输入保护，`10k × 1nF` 量级约 10us，用来抑制尖峰噪声。

一句话：**主驱动电压 M+ 通过 Q605B 受控接入分压采样；Q605A 用 5V 供电状态来使能采样；R609 提供弱诊断电流；R613/C606 负责 ADC 滤波。**



## IVV
![](assets/2026.5.26%20ASU2.5-TriC硬件电路/file-20260526174425586.png)


这个是**阀门电流检测电路**。

主电流路径是：

```text
venting_switch(约12V) -> R501(50mΩ采样电阻) -> CN507 pin1 -> 阀门线圈 -> CN507 pin2 -> VV
```

核心器件：

- `R501 50mΩ`：串在阀门供电线上，用来把电流变成一个很小的压差。
- `U501 CSA601LQ`：高边电流采样放大器，`IN+` 接 R501 上端，`IN-` 接 R501 下端，把 R501 上的压降放大后输出到 `ADC_IV`。
- `C505 2.2nF`：并在采样电阻两端，滤掉高频尖峰和开关噪声。
- `D505`：接在 `venting_switch` 和 `VV` 之间，按位置看是给阀门这种感性负载做**续流/钳位保护**，关断时给线圈电流一个释放路径。
- `CN507`：外接阀门连接器。
- `TP511/TP512/TP513`：测试点。

U501 料号 `CSA601LQCSOT235` 的增益是 **20V/V**，供电 `2.7V~5.5V`，共模输入范围 `-6V~40V`，所以它可以用 5V 供电，却检测 12V 高边电流。资料见 [AnalogySemi CSA601LQ](https://www.analogysemi.cn/product/linear/amplifier/current-sense-amplifier/588)。

换算关系很好记：

```text
Vshunt = Ivalve × 0.05Ω
ADC_IV ≈ Vshunt × 20
ADC_IV ≈ Ivalve × 1.0V/A
```

也就是说：

```text
阀门电流 1A -> ADC_IV 约 1V
阀门电流 2A -> ADC_IV 约 2V
阀门电流 3A -> ADC_IV 约 3V
```

一句话：**这是一个 12V 阀门驱动回路上的高边电流采样，MCU 通过 ADC_IV 判断阀门电流是否正常、是否开路、短路或卡滞。**

## 温度采样


![](assets/2026.5.26%20ASU2.5-TriC硬件电路/file-20260526174425585%201.png)这是一个**MOS 管附近的 NTC 温度采样电路**。

电路结构很简单：

```text
VCC_5V_1 -> R410 3.3k -> temp1 -> RT402 10k NTC -> GND
                         |
                         R413 1k -> ADC_MOS_TEMP1
                                  |
                                  C424 10nF
                                  |
                                 GND
```

怎么理解：

- `RT402 10k` 是 **NTC 热敏电阻**，靠近电机 MOS 放置，用来感知 MOS 区域温度。
- `R410 3.3k` 是上拉电阻，和 NTC 构成分压。
- `temp1` 这个点的电压送到 MCU 的 ADC。
- `R413 1k + C424 10nF` 是 ADC 输入低通滤波，抑制噪声。
- `C423 12pF` 是小电容，滤很高频的干扰，可能也用于 EMC。
- `TP429/TP430` 是测试点。

分压关系：

```text
Vtemp = 5V × R_NTC / (3.3k + R_NTC)
```

因为是 NTC：

```text
温度升高 -> RT402 阻值下降 -> Vtemp 下降 -> ADC 值下降
温度降低 -> RT402 阻值上升 -> Vtemp 上升 -> ADC 值上升
```

例如假设 25°C 时 NTC 是 10k：

```text
Vtemp = 5V × 10k / (3.3k + 10k)
      ≈ 3.76V
```

图里标的 `0.38mA` 也能对上：

```text
I = 5V / (3.3k + 10k) ≈ 0.376mA
```

一句话：**这是一个 5V 上拉的 NTC 分压测温电路，温度越高，ADC_MOS_TEMP1 电压越低，用来监测电机 MOS 附近温度。**

## 压力传感器


![](assets/2026.5.26%20ASU2.5-TriC硬件电路/file-20260526174425585.png)这是一个**压力传感器信号的上拉 + 滤波 + ADC 输入保护电路**。

可以按这个等效看：

```text
VCC_5V_2
   |
 R405 4.7k
   |
I_D_pressure_2 ---- R408 3.3k ---- ADC_PS_OUT2
   |                              |
 C413 10nF                       C414 10nF
   |                              |
  GND                            GND
```

各器件作用：

- `I_D_pressure_2`：压力传感器输出信号输入点。
- `R405 4.7k`：上拉到 5V。说明这个压力信号可能是**开漏/开集电极输出**，或者需要一个默认高电平。
- `C413 10nF`：输入端对地滤波，先把外部线束上的高频噪声压掉。
- `R408 3.3k`：ADC 串联电阻，限制进入 MCU ADC 管脚的电流，也和后级电容形成低通。
- `C414 10nF`：ADC 端滤波电容，让 `ADC_PS_OUT2` 更稳定。
- `TP410/TP411`：测试点。

如果外部传感器输出是开漏型：

```text
传感器断开/释放 -> R405 上拉 -> ADC_PS_OUT2 接近 5V
传感器拉低     -> ADC_PS_OUT2 接近 0V
```

如果外部传感器输出是模拟电压型，那么 `R405 4.7k` 会对传感器输出产生上拉偏置，软件读取的就是被这个上拉网络影响后的电压。

滤波大概是两级：

```text
第一级：R405 与 C413，抗线束干扰
第二级：R408 与 C414，保护 ADC 并继续低通滤波
```

一句话：**这是一个带 5V 上拉的压力信号采样输入，R408/C414 给 ADC 做限流和滤波，C413 用来抑制输入线上的噪声。**


## SV_AD_4 电流检测
![](assets/2026.5.26%20ASU2.5-TriC硬件电路/file-20260526174425583.png)


这个电路分两块：**SV_AD_4 电流检测** + **过流比较器保护**。

左边是电流采样放大：

```text
SV_AD_4 -> U716 CSA601N -> ADC_SV_C_4
```

- `U716 CSA601N` 是高边/低边电流采样放大器。
- `IN+ / IN-` 接在采样电阻两端，也就是检测 `SV_AD_4` 这一路电流。
- `OUT` 输出到 `ADC_SV_C_4`，给 MCU ADC 采样。
- `C753 220pF` 跨在输入两端，用来滤掉采样电阻上的高频尖峰。
- `C756 100pF` 在输出端对地，抑制输出高频噪声。

所以左边用途是：**软件读取这一路电流大小**。

右边是硬件过流判断：

```text
ADC_SV_C_4 -> R790 -> 比较器 U724 -
参考电压  -> R786/R807 -> 比较器 U724 +
U724 输出 -> SV_OVP4
```

这里有个阈值参考：

```text
Vref = 5V × R807 / (R790 + R807)
```

但图里实际是 `R790 = 2.2k` 从 ADC 输出过来，`R807 = 8.06k` 到地，这个节点同时接到比较器一路输入，所以它不是单纯固定参考，而是**对 ADC_SV_C_4 做了一个比例衰减**：

```text
V- ≈ ADC_SV_C_4 × 8.06k / (2.2k + 8.06k)
   ≈ ADC_SV_C_4 × 0.785
```

`R786 1M` 是比较器反馈电阻，给比较器做一点**迟滞**，避免临界点抖动。

`U724 RS331XF-Q1` 是比较器：

- 一路输入看电流采样信号。
- 另一路输入看阈值/反馈网络。
- 输出为 `SV_OVP4`，应是 “Solenoid Valve Over Voltage/Over Protection 4” 或 “Over Current Protect” 这类保护信号。

输出端：

```text
VCC_5V_1 -> R774 4.7k -> SV_OVP4
SV_OVP4 -> C740 680pF -> GND
```

说明比较器输出大概率是开漏/开集电极结构，需要 `R774` 上拉。`C740` 做输出去毛刺/抗干扰。

整体逻辑可以这样理解：

```text
正常电流：
U716 输出较低 -> 比较器输出保持正常状态

电流过大：
U716 输出升高 -> 比较器翻转 -> SV_OVP4 状态变化
```

一句话：**U716 把 SV_AD_4 的电流变成 ADC 电压给软件看；U724 同时做硬件过流比较，超过阈值就通过 SV_OVP4 给出保护/故障信号。**


### 有个疑问：
我没理解后面这个过流保护电路。按照虚短和虚断。同相输入的电压和反相输入的电压约等，同相输入的电压就是R807的电压，根据结点电压，应该是有两部分电压叠加？这是正相闭环还是反相闭环呢？
你这里把 **比较器** 当成 **运放闭环** 来分析了，所以会绕进去。

这个 U724 是 `RS331` 比较器，输出是开漏结构，不是线性运放闭环。这里**不能用虚短**，只能用“虚断”近似：输入端电流很小。

关键关系是：

```text
V+ = R790/R807 分压点
V- = ADC_SV_C_4
```

`R790 = 2.2k` 接 `VCC_5V_1`，`R807 = 8.06k` 接地，所以同相端基准电压约为：

```text
V+ = 5V × 8.06k / (2.2k + 8.06k)
   ≈ 3.93V
```

`R786 = 1M` 只是串到比较器同相输入。因为比较器输入电流很小，`R786` 上几乎没有压降，所以：

```text
U724 同相输入电压 ≈ R807 上端电压 ≈ 3.93V
```

它不是两部分电压叠加，也不是反馈闭环。

比较器判断逻辑是：

```text
ADC_SV_C_4 < 3.93V
=> V+ > V-
=> 比较器输出开漏关断
=> R774 上拉，SV_OVP4 = 高

ADC_SV_C_4 > 3.93V
=> V+ < V-
=> 比较器输出管导通
=> SV_OVP4 被拉低
=> 后级与门关断 MOS
```

所以这个过流保护的本质是：

```text
U716 输出电流检测电压
超过约 3.93V
U724 拉低 SV_OVP4
硬件禁止低边 MOS 继续导通
```

如果 U716 增益按 100 倍算，则过流阈值电流是：

```text
ADC_SV_C_4 = 100 × I × Rshunt

I_oc = 3.93V / (100 × Rshunt)
```

比如：

```text
Rshunt = 10mΩ -> I_oc ≈ 3.93A
Rshunt = 20mΩ -> I_oc ≈ 1.97A
Rshunt = 50mΩ -> I_oc ≈ 0.786A
```

一句话：**这是开环比较器阈值判断，不是正相闭环，也不是反相闭环；虚短不成立，只在翻转瞬间可以说两输入电压接近阈值相等。**

### 怎么区分放大器和比较器

区分方法看 **型号 + 外围连接 + 输出用途**，不要只看三角形符号，因为放大器和比较器画法很像。

**1. 看型号最直接**

例如你图里：

```text
U716 = CSA601N
```

`CSA` 一般是 Current Sense Amplifier，电流采样放大器。

```text
U724 = RS331
```

`331` 类似 LMV331/LM393 这一类，通常是比较器。

所以：

```text
U716 是放大器
U724 是比较器
```

**2. 看有没有负反馈**

运放/放大器常见是闭环工作：

```text
输出端 -> 电阻/网络 -> 反相输入端
```

这时可以近似用：

```text
V+ ≈ V-
输入电流 ≈ 0
```

也就是你说的“虚短、虚断”。

比较器一般没有负反馈，或者有一点正反馈做迟滞。它是开环判断：

```text
V+ > V-  -> 输出高/释放
V+ < V-  -> 输出低/拉低
```

这时**不能用虚短**。

**3. 看输出形式**

放大器输出是连续模拟量：

```text
输入 10mV -> 输出 1V
输入 20mV -> 输出 2V
输入 30mV -> 输出 3V
```

比较器输出是开关量：

```text
低电平 / 高电平
0 / 1
允许 / 禁止
正常 / 过流
```

你这张图里 U724 输出端有：

```text
R774 4.7k 上拉到 5V
C740 对地滤波
输出信号叫 SV_OVP4
```

这就是典型比较器开漏输出。它不是输出一个连续电压，而是拉低或释放。

**4. 看外围功能**

U716 这边：

```text
SV_AD_4 -> U716 -> ADC_SV_C_4
```

输出接 ADC，说明 MCU 要读一个连续电压，所以它是放大器。

U724 这边：

```text
ADC_SV_C_4 和阈值电压比较 -> SV_OVP4
```

输出去控制与门/保护逻辑，说明它是比较器。

一句话记忆：

```text
放大器：输出“有多大”，常接 ADC，可用虚短前提是负反馈闭环。
比较器：输出“过没过阈值”，常接保护/逻辑，不能用虚短。
```

在你这个电路里：

```text
U716：把电流采样电压放大，输出模拟量 ADC_SV_C_4
U724：判断 ADC_SV_C_4 是否超过阈值，输出保护信号 SV_OVP4
```

### 放大器放大的的是电压还是电流？
这里的放大器 **U716 放大的是电压，不是直接放大电流**。

更准确地说，它放大的是**采样电阻两端的压差**：

```text
Vshunt = V(IN+) - V(IN-)
```

而这个压差来自电流流过采样电阻：

```text
Vshunt = I × Rshunt
```

所以完整链路是：

```text
电流 I
  ↓ 经过采样电阻 Rshunt
产生小电压 Vshunt = I × Rshunt
  ↓ 被 U716 放大
输出 ADC 电压 Vout = Gain × Vshunt
```

因此：

```text
Vout = Gain × I × Rshunt
```

以 U716 增益 100 倍为例：

```text
Vout = 100 × I × Rshunt
```

比如采样电阻是 `10mΩ`：

```text
I = 1A
Vshunt = 1A × 0.01Ω = 10mV
Vout = 100 × 10mV = 1V
```

所以从器件角度讲：**它是电压放大器，放大输入压差**。  
从系统功能角度讲：**它是在测电流，因为输入电压代表电流大小**。

### 闭环体现在哪里？
放大器的**闭环**体现在：**输出端有一条反馈路径回到输入端，通常回到反相输入端 `-`**。

普通运放最典型是这样：

```text
        Rf
OUT ───/\/\/───┐
               |
              IN-
IN+ ───────── 运放
```

这条 `OUT -> IN-` 的路径就是**负反馈闭环**。有负反馈时，运放会自动调节输出，让：

```text
V+ ≈ V-
```

这时才可以用“虚短、虚断”。

你图里的 U716 是专用电流采样放大器，闭环主要在**芯片内部**，不是靠外部电阻搭出来的。它的外部只看到：

```text
IN+ / IN- / OUT / VCC / GND
```

但芯片内部已经做好了反馈电阻网络，所以它能稳定输出：

```text
Vout = Gain × (VIN+ - VIN-)
```

也就是：

```text
ADC_SV_C_4 = 100 × Vshunt
```

它不是裸运放，所以你在外围看不到像 `Rf/Rin` 那样完整的闭环电阻网络。增益 100 倍也是内部固定好的。

和 U724 对比就更清楚：

```text
U716：内部闭环放大器，输出是连续模拟电压
U724：比较器，开环比较，输出只有高/低状态
```

U724 那里没有从输出 `SV_OVP4` 回到输入端的负反馈路径，所以不能用虚短。`R774` 只是输出上拉，`C740` 是滤波；它们都不是闭环反馈。

一句话：**普通运放的闭环看外部反馈电阻；U716 这种专用采样放大器的闭环藏在芯片内部；比较器 U724 没有负反馈闭环，所以不能按虚短分析。**

### 我还是不能理解为什么放大器能用虚短和虚断但是比较器不行？主体的硬件不是一样的吗

不是主体长得像就都能用“虚短”。关键不是器件外形，而是**工作状态**。

先分开两个概念：

```text
虚断：输入端电流很小
虚短：两个输入端电压近似相等
```

**虚断**来自输入阻抗高，所以放大器、比较器很多时候都可以近似认为：

```text
I+ ≈ 0
I- ≈ 0
```

但**虚短不是器件天然属性**。虚短是**负反馈闭环强迫出来的结果**。

比如运放开环关系是：

```text
Vout = A × (V+ - V-)
```

其中 `A` 很大，可能几十万倍。如果没有反馈，只要：

```text
V+ - V- = 1mV
```

理论输出就想变成几百伏，实际做不到，于是直接饱和到高电平或低电平。这就是比较器的状态。

所以比较器判断的是：

```text
V+ > V-  -> 输出高/释放
V+ < V-  -> 输出低/拉低
```

它不会主动让 `V+ ≈ V-`。

而运放闭环时，比如电压跟随器：

```text
OUT 接回 V-
输入接 V+
```

如果 `V-` 比 `V+` 低一点，运放输出就升高；输出升高又把 `V-` 拉高。  
如果 `V-` 比 `V+` 高一点，运放输出就降低；输出降低又把 `V-` 拉低。

最后系统自动稳定在：

```text
V- ≈ V+
```

这才叫虚短。

所以一句话：

```text
虚短 = 高增益 + 负反馈 + 没有饱和
```

缺一个都不行。

你图里的两个器件：

```text
U716：电流采样放大器
```

它内部有负反馈网络，芯片设计目标就是让它在线性区工作，输出连续电压：

```text
Vout = Gain × (VIN+ - VIN-)
```

所以它可以当“放大器”理解。

```text
U724：比较器
```

它没有负反馈闭环，输出只是在高/低之间切换。它的任务不是放大出一个连续比例电压，而是判断：

```text
ADC_SV_C_4 是否超过基准电压
```

所以 U724 这里不能用虚短，只能说在**翻转临界点附近**：

```text
V+ ≈ V-
```

但那只是“刚好到阈值”，不是比较器一直强迫两个输入相等。

硬件主体确实有相似处：都有差分输入级，都会比较 `V+ - V-`。区别是：

```text
运放：为线性闭环放大优化
比较器：为快速翻转、饱和开关输出优化
```

所以记住这个判断法就够了：

```text
有负反馈并在线性区 -> 可以用虚短、虚断
没有负反馈，输出高低跳变 -> 只能用虚断，不能用虚短
```





## 三腔阀低边驱动
![](assets/2026.5.26%20ASU2.5-TriC硬件电路/file-20260527085155098.png)
这段是 **MOS 低边驱动链路**，不是采样电路。

信号流向大概是：

```text
MCU/逻辑信号 SV_LSD_4
   |
 R763 10k
   |
 输入滤波节点
   |
 U709 74HCT1G08 与门
   |
 R764 10Ω
   |
 Q703B MOS 栅极
```

各部分作用：

- `SV_LSD_4`：低边驱动控制信号，应该来自 MCU 或逻辑控制。
- `R763 10k + C725 10nF`：输入 RC 滤波/延时，滤掉毛刺。时间常数约：
  
  ```text
  τ = 10k × 10nF = 100us
  ```

- `U709 74HCT1G08`：单路二输入与门。  
  一个输入接 `SV_LSD_4` 滤波后的信号，另一个输入接 `SV_OVP4`。所以它实现硬件联锁：

  ```text
  OUT = SV_LSD_4 AND SV_OVP4
  ```

- `SV_OVP4`：上一张图里比较器输出的保护信号。它参与与门控制，过流/异常时可以禁止 MOS 打开。
- `R800 10k`：输出下拉，保证 U709 输出悬空或上电未稳定时，MOS 栅极不会乱开。
- `R764 10Ω`：MOS 栅极电阻，限制栅极充放电瞬态电流，减小振铃和 EMI。
- `Q703B VDM6E6D120LSA`：低边 MOS。栅极高电平时导通，把负载下端拉到地；栅极低电平时关断。

正常逻辑：

```text
SV_LSD_4 = 1 且 SV_OVP4 = 1
-> U709 输出高
-> MOS 栅极高
-> Q703B 导通
-> 负载被低边拉地

SV_LSD_4 = 0 或 SV_OVP4 = 0
-> U709 输出低
-> MOS 关断
```

一句话：**MCU 的低边驱动信号经过 RC 滤波后，还要经过过流保护信号 SV_OVP4 允许，最后才驱动 MOS 栅极；保护触发时硬件会切断 MOS。**


## 电流回采的下拉电阻作用
![](assets/2026.5.26%20ASU2.5-TriC硬件电路/file-20260528092804730.png)
这里的下拉电阻是 `R727 2.2k`，它和 `R717 15k` 组成了一个**分压 + 默认下拉**网络。

结构是：

```text
ADC_SV_CS_A ---- R717 15k ---- CS脚
                              |
                             R727 2.2k
                              |
                             GND
```

主要作用有三个：

1. **把电流回采电压分压到芯片 CS 可接受范围**

CS 脚电压约为：

```text
V_CS = ADC_SV_CS_A × R727 / (R717 + R727)
     = ADC_SV_CS_A × 2.2k / (15k + 2.2k)
     ≈ ADC_SV_CS_A × 0.128
```

也就是约 **1/7.8**。

例如：

```text
ADC_SV_CS_A = 5V -> CS ≈ 0.64V
ADC_SV_CS_A = 3V -> CS ≈ 0.38V
```

2. **给 CS 脚一个确定的低电平**

如果前级 `ADC_SV_CS_A` 断开、上电未初始化、或者处于高阻态，`R727` 会把 CS 脚拉到地，避免 CS 悬空误判。

3. **和 R717/C701 做抗干扰/限流**

`R717` 限制进入 CS 脚的电流，`R727` 降低节点阻抗，提高抗噪声能力。  
左边的 `C701 10nF` 是对 `ADC_SV_CS_A` 节点滤波。

一句话：**R727 不是单纯“拉低”，它主要和 R717 一起把电流回采电压缩小到 CS 脚能用的范围，同时保证 CS 脚不悬空。**
