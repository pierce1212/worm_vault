

# EcuMCommonConfiguration
<mark style="background: #FF5582A6;">`EcuMCommonConfiguration`，主要配置 EcuM 和 OS 的连接关系。</mark>
![](assets/2026.06.24%20bsw中ISOLAR配置--EcuM/file-20260624153407712.png)
## CommonConfiguration 中的 OS Resource 是什么？

`EcuMOSResource = RES_AUTOSAR_ECUM_CORE0 / CORE1`

这是给 EcuM 用的 OS Resource。你的工程是多核，所以每个 core 配一个独立 resource：
core 0 -> RES_AUTOSAR_ECUM_CORE 0
core 1 -> RES_AUTOSAR_ECUM_CORE 1
<mark style="background: #FFF3A3A6;">StatusType EcuM_Prv_GetResource(CoreIdType core_ID)</mark>
这个 resource 主要用于 EcuM 进入 sleep / wakeup restart 这类关键流程时加锁，避免其他 OS task 打断 EcuM 的 sleep 切换过程。

## 拓展：spinlock 和 Resource 有什么区别

简单说：

`Resource` 主要管 **同一个 core 内的任务调度互斥**。  
`Spinlock` 主要管 **多个 core 之间的共享数据互斥**。

对比一下：

|项目|OS Resource|Spinlock|
|---|---|---|
|典型 API| `GetResource()` / `ReleaseResource()` | `GetSpinlock()` / `ReleaseSpinlock()` |
|主要用途|防止本 core 上其他 task/ISR 打断关键流程|防止其他 core 同时访问同一份共享数据|
|等待方式|一般不是忙等，OS 通过优先级天花板/调度规则保护|忙等，拿不到锁就在 CPU 上转|
|适用范围|通常是 core-local 调度保护|cross-core 互斥|
|持锁时间|可以比 spinlock 稍长，但仍应短|必须非常短，不能做耗时操作|
|典型风险|忘记释放会影响本 core 调度|忘记释放会卡住其他 core，严重时系统挂死|
|是否适合睡眠流程|适合保护 EcuM sleep sequence 不被本 core task 打断|不适合包住长流程|

你工程里这两个都有。

你前面看到的 `EcuMOSResource`：

```
RES_AUTOSAR_ECUM_CORE0
RES_AUTOSAR_ECUM_CORE1
```

生成到这里：

```
case 0:
    GetResource(RES_AUTOSAR_ECUM_CORE0);

case 1:
    GetResource(RES_AUTOSAR_ECUM_CORE1);
```

[EcuM_Cfg_Sleep.c (line 45)](/E:/github/ECAS_qirui_Single_Chamber/BasicSoftware/src/bsw/EcuM/EcuM_Cfg_Sleep.c:45)

这个是 EcuM 进入 sleep / wakeup restart 时用的 **OS Resource**。它保护的是 EcuM 的睡眠切换大流程，防止本 core 上其他任务把流程打断。

但你工程里的 `SchM_Enter_EcuM()` 是另一回事，它映射到了 Spinlock：

```
#define SchM_Enter_EcuM_EcuM()  ( (void)GetSpinlock(EcuM) )
#define SchM_Exit_EcuM_EcuM()   ( (void)ReleaseSpinlock(EcuM) )
```

也就是说，EcuM 内部那些很短的全局变量保护，比如 pending/validated wakeup event 这些共享状态，走的是 **Spinlock**。这是为了多核之间同步。

可以这样理解：

```
Resource：
保护“这一段 EcuM sleep 流程别被本 core 调度打断”

Spinlock：
保护“这个 EcuM 全局变量别被另一个 core 同时改”
```

所以在 `EcuM_Prv_GoSleepSequence()` 里拿 `RES_AUTOSAR_ECUM_CORE0/1` 是合理的，因为它包的是较大的 sleep sequence。  
而在 `EcuM_SetWakeupEvent()`、`EcuM_MainFunction()` 里读写 `EcuM_Prv_dataPendingWakeupEvents_u32` 这种共享状态，用 `SchM_Enter_EcuM()` / spinlock 更合理。

```
ECU 已经准备进入这个 SleepMode 之后，

如果 ECU 处于这个 EcuMSleepMode，
并且 CAN transceiver / CanIf 后面上报了 CAN wakeup，

```
