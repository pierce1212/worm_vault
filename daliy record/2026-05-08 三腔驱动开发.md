## PWM疑问
1. 什么是UC?
eMIOS的通道，通道又叫Unified channels（UC）
2. 什么是Counter Bus呢？
在传统PWM生成模块中，里面都会有一个内部计数器，如果让其生成想要的PWM波则需要设置这个内部计数器的计数方式以及周期等，而eMIOS的Counter Bus则就是代替了这个内部计数器。而这个Counter Bus信号则就是另外一个UC的内部计数器CNT（每个UC内部都有一个内部计数器CNT）

4. UC时钟源选择有什么区别？
![](assets/2026-05-08%20三腔驱动开发/file-20260527144123664.png)
	  EMIOS_PWM_IP_PS_SRC_MODULE_CLOCK

    含义：

    使用 eMIOS 模块原始时钟作为该通道 prescaler 的输入。

    链路大概是：

    McuClockForPwm / eMIOS module clock
        ↓
    Channel internal prescaler
        ↓
    Channel counter / PWM period

    也就是说，该通道直接拿 eMIOS 模块时钟，再经过本通道自己的分频。
	频率较高
    PWM 分辨率更高
    更适合需要精细占空比/周期控制的 PWM
    默认值通常是这个
    如果目标 PWM 频率不低，一般优先用这个
        McuClockForPwm = 80 MHz
    
    Channel prescaler = 80

    则该通道 tick 频率：

    80 MHz / 80 = 1 MHz

    如果 PWM 目标 200 Hz：

    Default Period = 1 MHz / 200 = 5000 ticks
 5. EMIOS_PWM_IP_PS_SRC_PRESCALED_CLOCK

    含义：

    使用 eMIOS 模块已经预分频后的时钟作为该通道 prescaler 的输入。

    链路大概是：

    McuClockForPwm / eMIOS global prescaler
        ↓
    Prescaled clock
        ↓
    Channel internal prescaler
        ↓
    Channel counter / PWM period

    也就是说，它不是直接用模块原始时钟，而是先经过 eMIOS 全局分频，再进入通道自己的分频。

    特点：

    输入时钟更低
    可实现更长周期/更低频 PWM
    但分辨率会降低
    适合低频 PWM，例如几十 Hz、几 Hz 这种
    如果 Default Period 算出来超过最大值，可以考虑切到这个

    例如：
    Channel prescaler = 80

    如果选 MODULE_CLOCK：

    F_tick = 80 MHz / 80 = 1 MHz

    如果选 PRESCALED_CLOCK：

    F_tick = 80 MHz / 80 / 80 = 12.5 kHz

    同样目标 200 Hz：

    MODULE_CLOCK 时：

    Default Period = 1 MHz / 200 = 5000

    PRESCALED_CLOCK 时：

    Default Period = 12.5 kHz / 200 = 62.5

    这时周期 tick 数变小了，分辨率也变粗了。




5. master bus 是什么？
    在 NXP S32K3 的 eMIOS 里，Master Bus 可以理解成：

    给一组 eMIOS 通道提供“公共时间基准”的主计数器通道。

    也就是说，它本身占用一个 eMIOS channel，但它通常不是直接输出 PWM，而是作为“总线/时间基准”给其它 PWM 通道使用。

    简单理解：

    Master Bus = 公共计数器 / 公共周期源 / 时间基准通道

    其它 PWM 通道可以引用它，用它的计数周期来产生 PWM。

 7. Emios通道选用全局的counter bus(master bus)后，通道内的分频系数还起作用吗？
通道选用了全局 Master Bus / BUS_A 之后，PwmEmiosChannels_SVLSD2 这个 CH_18 通道里的 Clock prescaler = DIV_4 对 PWM 周期频率基本不起作用。 周期频率由 **eMIOS_0 模块时钟、eMIOS common 全局分频、Master Bus CH23 分频、Master Bus period** 决定.
PWM频率 = eMIOS_0模块频率 / eMIOS common分频 / MasterBusPrescaler / MasterBusPeriod
在 OPWMB + BUS_A 下，真正决定周期的是 **MCL 里的 MasterBus_A period**，不是这个 PWM 页面的 Default Period。PWM 页面的 Default Period=200 更多是给 PWM 模块配置一致性、默认参数、100% duty 判断等使用。

```
Mcl / MasterBus_A:
Prescaler = DIV_4
Default period = 200

Pwm / PwmChannel:
Default Period = 200
Default Duty cycle = 16384  // 50%
```

### ADC 疑问

1. ADC PIT模块最好选什么时钟频率
	1. PIT
    - 你图里选的 AIPS_SLOW_CLK = 40 MHz
    - 这个一般是合理的，PIT 用来做周期中断、OS tick、超时计时，40 MHz 通常没问题
    - 如果你没有特殊低功耗要求，PIT 这项基本可以保持不动

    2. ADC
    - 不建议直接选 CORE_CLK = 160 MHz
    - ADC 最好用“稳定的外设/专用ADC时钟源”，再分频到满足 ADC 规格的频率
    - 如果只能在这个界面里选参考时钟，优先选 PLL/FIRC/FXOSC 这类稳定源，不要直接绑 CPU 主频

    更实用的建议：
    - PIT：AIPS_SLOW_CLK，40 MHz，OK
    - ADC：最终 ADC 时钟尽量落在“手册允许上限以下”，工程上常见做法是 20~40 MHz 更稳妥
    - 如果你的 ADC 采样率要求不高，偏低一点更安全
    - 如果追求吞吐量，就按芯片手册的 ADC 最大时钟来定，不要凭感觉拉到 160 MHz

    你这张图里最明显的问题就是：
    - PIT 没啥问题
    - ADC 选 CORE_CLK 160 MHz 风险很大，建议改
